# Uber Eats Login Clone — Supabase + Vercel

A pixel-perfect recreation of the Uber Eats login flow with real authentication.

## Files
- `index.html` — Complete login UI (3 screens + dashboard)
- `vercel.json` — Vercel deployment config
- `README.md` — This guide

---

## Step 1: Create Supabase Project

1. Go to [https://supabase.com](https://supabase.com) and sign up
2. Click **New Project** → fill in name + password → Create
3. Go to **Settings → API**
4. Copy:
   - **Project URL** → looks like `https://abcxyz.supabase.co`
   - **anon public key** → long JWT string

---

## Step 2: Enable Email Auth in Supabase

1. In your project → **Authentication → Providers**
2. Enable **Email** provider
3. (Optional) Disable "Confirm email" for easier testing

---

## Step 3: Add Your Supabase Keys to index.html

Open `index.html` and find these two lines near the bottom:

```js
const SUPABASE_URL  = 'https://YOUR_PROJECT_ID.supabase.co';
const SUPABASE_ANON = 'YOUR_ANON_PUBLIC_KEY';
```

Replace them with your actual values.

---

## Step 4: Create a Students Table (For Extra Marks)

In Supabase → **SQL Editor** → run:

```sql
CREATE TABLE students (
  id         bigint generated by default as identity primary key,
  email      text unique not null,
  created_at timestamp default now()
);

-- Allow logged-in users to insert their own row
ALTER TABLE students ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can insert own row" ON students
  FOR INSERT WITH CHECK (auth.email() = email);
CREATE POLICY "Users can view own row" ON students
  FOR SELECT USING (auth.email() = email);
```

---

## Step 5: Upload to GitHub

1. Create a new repo on [https://github.com](https://github.com)
2. Upload `index.html`, `vercel.json`, `README.md`

---

## Step 6: Deploy to Vercel

1. Go to [https://vercel.com](https://vercel.com) → Log in with GitHub
2. Click **Add New Project** → Import your GitHub repo
3. Click **Deploy**
4. Your site is live at `https://your-project.vercel.app` 🎉
